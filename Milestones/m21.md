# KnowWhy Milestone M21 — Deployment, DevOps & Infrastructure

## ROLE

You are the Lead DevOps Engineer and Cloud Architect responsible for deploying KnowWhy to production.

Continue from Milestone M20.

Assume Milestones M01–M20 are complete.

Do NOT recreate previous work.

Extend the existing architecture.

---

# PROJECT CONTEXT

KnowWhy is now production-ready from a software perspective.

The next objective is to make it deployable, scalable, observable, and maintainable in real-world cloud environments.

Deployment should be automated, repeatable, secure, and environment-independent.

---

# OBJECTIVE

Implement a complete DevOps and Infrastructure pipeline for KnowWhy.

The platform should support local development, staging, and production deployments.

---

# SCOPE

- Production Docker Images
- Docker Compose (Production)
- Environment Management
- CI/CD Pipeline
- Reverse Proxy
- HTTPS
- Monitoring
- Logging
- Backup Strategy
- Deployment Documentation

Do NOT implement:

- New application features
- Research documentation

# CONTAINERIZATION

Review and optimize:

- Frontend Dockerfile
- Backend Dockerfile
- PostgreSQL
- Redis
- Worker Services

Create optimized production images.

Reduce image size where possible.

---

# ENVIRONMENT MANAGEMENT

Support environments:

- Development
- Testing
- Staging
- Production

Manage configuration securely.

Do not hardcode secrets.

Use environment variables and secret management.

---

# CI/CD

Implement GitHub Actions workflows for:

- Linting
- Testing
- Security Scan
- Build
- Docker Image Creation
- Deployment (optional/manual approval)

Prevent deployment if tests fail.

---

# REVERSE PROXY

Configure:

- Nginx (preferred) or Traefik

Responsibilities:

- HTTPS
- Static asset serving
- API routing
- Compression
- Security headers

---

# HTTPS

Support TLS using Let's Encrypt or equivalent.

Redirect HTTP to HTTPS.

Secure cookies.
