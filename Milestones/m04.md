# PROJECT CONTEXT

KnowWhy is a multi-tenant SaaS platform.

Every authenticated user belongs to one or more Organizations.

An Organization is the top-level entity that owns:

- Members
- Projects
- Integrations
- Organizational Memory
- AI Data

Nothing inside KnowWhy exists outside an Organization.

This milestone establishes the multi-tenant foundation that every future feature will rely on.

# MEMBERSHIP MODEL

Create a Membership table linking Users and Organizations.

Fields

- User ID
- Organization ID
- Role
- Joined At

A user may belong to multiple organizations.

---

# ROLES

Implement role-based access control (RBAC).

Roles:

- Owner
- Admin
- Member

Permissions

### Owner

- Delete Organization
- Manage Members
- Change Roles
- Manage Integrations
- Transfer Ownership

### Admin

- Invite Members
- Remove Members
- Manage Projects (future)
- View Integrations

### Member

- Access Workspace
- View Projects
- Use AI Features
- Update Own Profile

Design the system so new roles can be added easily.
