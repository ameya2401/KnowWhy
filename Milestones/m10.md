# PROJECT CONTEXT

KnowWhy now imports information from multiple platforms.

However, each integration stores data independently.

The objective of this milestone is to create a unified internal knowledge representation.

This milestone is NOT AI.

This milestone is NOT embeddings.

This milestone is NOT vector search.

This milestone builds the foundation that every AI feature will use.

Think of this as KnowWhy' internal brain.

# NORMALIZATION ENGINE

Implement a reusable Normalization Engine.

Responsibilities:

- Convert provider-specific data into KnowledgeItems.
- Preserve original metadata.
- Eliminate duplicate entities.
- Maintain source references.
- Track updates.

GitHub, Notion, and Google Drive must all use this engine.
