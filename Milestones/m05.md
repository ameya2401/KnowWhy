# PROJECT CONTEXT

KnowWhy organizes work using Organizations and Projects.

Every Project belongs to exactly one Organization.

Projects act as the primary workspace where future features will attach:

- GitHub Repositories
- Notion Pages
- Google Drive Files
- Calendar Events
- AI Memory
- Search Index
- Conversations

This milestone establishes the Project Management system.

- UUID
- Organization ID
- Name

- Description
- Visibility (Private/Public)
- Status (Active/Archived)
- Color
- Icon

# PROJECT MEMBERS

Implement a ProjectMember model.

Fields:

- User ID
- Project ID
- Role
- Joined At

A user may belong to multiple projects.

---

# PROJECT ROLES

Implement project-level RBAC.

Roles:

- Owner
- Maintainer
- Contributor
- Viewer

Permissions

### Owner

- Delete project
- Manage settings
- Manage members
- Archive project

### Maintainer

- Update project
- Invite members
- Manage integrations (future)

### Contributor

- Access project
- Use AI
- View integrations

### Viewer

- Read-only access
