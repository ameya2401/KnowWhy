# PROJECT CONTEXT

KnowWhy currently supports:

- Keyword Search
- Metadata Search
- Vector Embeddings

However, neither approach alone is sufficient.

Keyword search misses semantic meaning.

Semantic search sometimes ignores exact technical terms.

The objective of this milestone is to combine both approaches into a production-ready Hybrid Search Engine.

This becomes the retrieval layer that every future AI feature will use.

---

# OBJECTIVE

Implement a Hybrid Search Engine capable of retrieving the most relevant organizational knowledge using both lexical and semantic techniques.

The system should prioritize accuracy, explainability, and performance.

# HYBRID SEARCH ARCHITECTURE

Implement the following retrieval pipeline:

```
User Query

↓

Query Processor

↓

Keyword Search

+

Semantic Search

↓

Result Fusion

↓

Re-ranking

↓

Deduplication

↓

Top-K Results

↓

Future RAG Pipeline
```

Every search request must pass through this pipeline.

# RESULT FUSION

Combine lexical and semantic results.

Implement Reciprocal Rank Fusion (RRF) as the default strategy.

Architecture should allow future ranking strategies.

Avoid duplicate results.

---

# RE-RANKING

Re-rank results using weighted scoring.

Consider:

- Keyword relevance
- Semantic similarity
- Source reliability
- Document freshness
- Organization relevance
- Project relevance

Scoring weights should be configurable.
