# Figma Checklist

Version: 1.0

Status: Pending

---

# Purpose

This document defines everything that must be designed in Figma before development begins.

The objective is not pixel-perfect designs.

The objective is to eliminate uncertainty during implementation.

---

# Design Goals

The Figma prototype should:

- Define the complete user interface.
- Demonstrate all major user flows.
- Reuse components wherever possible.
- Be ready for frontend implementation.

# Design Review

Before implementation, confirm:

- [ ] Every screen is designed.
- [ ] Every navigation path works.
- [ ] Components are reusable.
- [ ] No placeholder screens remain.
- [ ] Design matches the MVP scope.

---

# Definition of Done

The design phase is complete when:

- All MVP screens are designed.
- A clickable prototype exists.
- The component library is complete.
- The design system is consistently applied.
- Developers can begin implementation without guessing UI behavior.
